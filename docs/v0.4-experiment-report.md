# Krystal Quorum v0.4 Candidate Experiments

Branch: `experiment/v0.4-candidates`

This branch tests larger v0.4 ideas before they are promoted into the shipped
CLI path. Prototype code lives under `experiments/`; promoted product code
lives under `src/krystal_quorum/`.

## Candidates Tested

### Retry malformed reviewer output

Prototype: `try_repair_parse`

Product status: promoted into the command, Ollama, and OpenAI-compatible
reviewer adapters on this branch.

The retry probe simulates a reviewer producing malformed output first, then a
valid strict JSON response on a later attempt. It preserves failed raw attempts,
sets the final `ReviewerOutput.retries` count, and only retries parse failures,
not intentional reviewer abstentions.

Decision read: low-risk. The product path now retries malformed reviewer output
once, adds a JSON-only reminder on the retry prompt, and keeps failed raw output
visible in artifacts.

### Consensus matching upgrade

Prototype: `candidate_issue_clusters`

Product status: promoted into reconciliation as schema `1.2`.

The product matcher groups reviewer issues by deterministic concept terms such
as `rollback`, `tests`, `acceptance`, `security`, `dependencies`, and
`observability`. It successfully groups "rollback" and "backout" as the same
review signal while keeping unrelated rollback and security findings separate.

Decision read: promoted. Deterministic concept matching is safer than opaque
semantic matching for v0.4. The product path now persists `issue_clusters` with
members, direct match edges, and match reasons. Set
`KRYSTAL_QUORUM_CONSENSUS_MATCHER=legacy` to temporarily restore the older
token-overlap grouping behavior.

Implementation note: generic section text such as `Plan` is excluded from
support-term overlap so it cannot make unrelated findings look like consensus.

### Evidence-derived confidence

Prototype: `candidate_confidence`

The prototype derives confidence from observable signals: reviewer
participation, diversity status, shared blockers, singleton blockers,
contradictions, and round-2 changes. It intentionally ignores model-reported
self-confidence.

Decision read: worth pulling in after consensus matching. It will be more
meaningful once shared issue detection is stronger.

### Opinionated review rubric

Prototype: `evaluate_rubric`

The prototype scans a plan for explicit coverage of:

- acceptance criteria
- rollback plan
- tests and verification
- security risk
- dependency scope
- observability plan

Core preflight sections are marked `UNSATISFIED` when missing. Contextual
sections such as security, dependencies, and observability are marked `UNCLEAR`
when not explicit.

Decision read: high brand value and likely the strongest product differentiator.
It should become part of the reviewer prompt and/or a pre-LLM deterministic
preflight, but the wording should be tuned against real Krystal plans before a
release.
